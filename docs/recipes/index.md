# Recipes

Recipes are end-to-end guides that show you how to combine Habits tools to solve a real problem. Each recipe is built around a specific goal and walks you through every step, from setup to a live running system.

Pick the recipe that matches your situation.

## Available Recipes

| Recipe | Who it's for |
|---|---|
| [Company Automations Hub](./company-hub) | Teams that need multiple habits hosted under one roof with an admin panel, custom subdomains, and HTTPS |
| [No-Code Automation Builder](./no-code-automation) | Builders who want to create visual workflows, connect APIs and AI, and deploy them without writing backend code |
| [Native Apps](./native-apps) | People who want to run habits on their phone (iOS / Android) or as a native desktop app (Mac, Windows, Linux) |
| [Build for Customers](./build-for-customers) | Developers who want to package habits as a branded SaaS product their customers can access through a URL |

## How recipes differ from the docs

The [tools docs](/tools/) explain what each individual tool does. Recipes show you **how to combine tools** to achieve a specific outcome. A recipe will tell you which tool to reach for at each step and why.
